# MyAsset Portfolio 사용설명서

이 문서는 현재 저장소 기준의 실제 운영 흐름만 요약합니다.
예전 PostgreSQL/별도 DB 컨테이너 기준 안내는 더 이상 유효하지 않습니다.

## 이 프로젝트로 할 수 있는 일

- 자산, 거래, 환율, 현금흐름 관리
- 카드/은행 알림 기반 지출 수집
- 뉴스/경제/e스포츠 브리핑 수집
- 텔레그램 알림 및 AI 리포트 생성
- 개인용 자동매매 엔진 실험

## 기본 구조

현재 서비스는 Docker Compose 기준으로 나뉘어 있습니다.

- `backend-api`: 메인 웹 API
- `alarm-collector`: 스마트폰 알림 수집
- `news-scheduler`: 뉴스/알림/운영 스케줄링
- `trading-scheduler`: 트레이딩 엔진 스케줄링
- `esports-monitor`: e스포츠 상태 감시
- `sync-prices`: 시세/리포트 보조 동기화

데이터 저장은 기본적으로 `backend/storage/db/portfolio.db` SQLite 파일을 사용합니다.

## 처음 시작할 때

### 1. 환경 파일 준비

```bash
cp backend/.env.example backend/.env
cp backend/.env.secrets.example ~/ai-models/myasset.secrets.env
```

정리:
- `backend/.env`: repo 안에 둬도 되는 일반 설정
- `~/ai-models/myasset.secrets.env`: API 키, 토큰, 비밀값

### 2. 실행

```bash
docker compose up -d
```

### 3. 접속

- 웹 UI: `http://localhost:5173`
- API 문서: `http://localhost:8000/docs`

## 로그인 방식

기본 로그인:
- 네이버 OAuth 로그인
- 성공 시 쿠키 세션 사용

보조 로그인:
- 비상용 `API_TOKEN` 직접 입력

즉 일반 사용은 네이버 로그인, 장애 대응은 API 토큰이라고 보면 됩니다.

## 주요 기능 사용법

### 자산 관리

- 종목/현금/기타 자산을 등록하면 대시보드에서 평가 금액을 확인할 수 있습니다.
- 거래 내역과 환율 기준값을 함께 관리할 수 있습니다.
- 벤치마크 이름과 수익률을 직접 넣어 비교용으로 사용할 수 있습니다.

### 지출 기록

- 모바일 알림을 `alarm-collector`로 보내면 카드/은행 사용 내역이 자동 수집됩니다.
- 수집된 내역은 분류 규칙과 모델을 통해 카테고리화됩니다.
- 필요하면 CSV/엑셀 업로드로 일괄 반영할 수 있습니다.

### 뉴스/브리핑

- 뉴스 수집은 `news-scheduler`가 담당합니다.
- 경제/기술/환율/e스포츠 관련 데이터가 저장되고, 일부는 텔레그램으로 브리핑됩니다.
- 뉴스 통계성 조회는 현재 SQLite + DuckDB 기반으로 처리됩니다.

### 자동매매 엔진

- `trading-scheduler`가 단타/스윙 실험용 전략을 주기적으로 실행합니다.
- 현재는 개인 소액 계좌 실험 전제이며, 범용 사용자용 기능으로 문서화하지 않습니다.
- 관련 산출물은 주로 `backend/storage/trading_engine/` 아래에 남습니다.

## 자주 쓰는 운영 명령

```bash
# 전체 상태 확인
docker compose ps

# 메인 API 로그
docker compose logs -f backend-api

# 뉴스 스케줄러 로그
docker compose logs -f news-scheduler

# 트레이딩 스케줄러 로그
docker compose logs -f trading-scheduler

# e스포츠 모니터 로그
docker compose logs -f esports-monitor

# 한 서비스만 재시작
docker compose restart backend-api
docker compose restart trading-scheduler
```

## 점검 포인트

### 서버가 안 뜰 때

```bash
docker compose logs -f backend-api
docker compose logs -f news-scheduler
```

대부분은 환경변수 누락, 외부 API 키 누락, LLM 서버 미기동 문제로 이어집니다.

### 로그인 안 될 때

- `NAVER_CLIENT_ID`, `NAVER_CLIENT_SECRET` 확인
- Redirect URI 설정 확인
- 프런트의 `serverUrl`이 실제 백엔드를 가리키는지 확인

### KIS 연동 안 될 때

- `KIS_MY_APP`, `KIS_MY_SEC` 등 시크릿 존재 여부 확인
- `KIS_CONFIG_DIR` 마운트 확인
- `backend-api` 또는 `trading-scheduler` 로그에서 인증/호가 오류 확인

### 텔레그램 알림 안 올 때

- `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID` 확인
- 관련 스케줄러 로그 확인

## 데이터 위치

- 메인 DB: `backend/storage/db/portfolio.db`
- 런타임 데이터: `backend/data/`
- 로그: `backend/logs/`
- 트레이딩 엔진 상태/출력: `backend/storage/trading_engine/`

## 스크립트 해석법

- `backend/scripts/runners`: 현재 실행 경로에서 직접 쓰는 러너
- `backend/scripts/maintenance`: 운영 중 보정/정리용
- `backend/scripts/db_setup`, `migrations`: DB 관련 초기화/마이그레이션
- `backend/scripts/legacy`: 가끔 수동 실행하는 실험/보조/진단 스크립트

중요:
- `legacy`는 곧바로 삭제 대상이라는 뜻이 아닙니다.
- 다만 기본 운영 경로는 아니므로, 자동 스케줄이나 메인 서비스와 동일하게 가정하면 안 됩니다.
