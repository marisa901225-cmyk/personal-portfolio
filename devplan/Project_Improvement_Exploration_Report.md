**기대 효과:** 민감정보 유출 리스크 감소, 운영 로그의 안전성/컴플라이언스 향상.

| 항목 | 내용 |
|------|------|
| **ID** | `env-compat-telegram-llm` |
| **카테고리** | 🧹 코드 품질 |
| **복잡도** | Low |
| **대상 파일** | backend/routers/telegram_webhook.py, backend/integrations/telegram.py, backend/.env.example |

**현재 상태:** 텔레그램 웹훅/전송 모듈이 사용하는 ENV(`ALARM_TELEGRAM_CHAT_ID`, `ALARM_TELEGRAM_BOT_TOKEN`, `X_TELEGRAM_BOT_API_SECRET_TOKEN`)와 `.env.example`의 키(`TELEGRAM_CHAT_ID`, `TELEGRAM_BOT_TOKEN`)가 불일치하여 초기 설정 시 오동작(무제한 허용/차단/전송 실패) 가능성이 있음. LLM 원격 모드(`LLM_BASE_URL`, `LLM_REMOTE_DEFAULT_MODEL` 등)도 예시가 부족함.  
**개선 내용:** 라우터에서 기존 키(예: `TELEGRAM_CHAT_ID`)를 fallback으로 지원하고, `.env.example`에 실제 사용되는 키 및 원격 LLM 관련 키를 명확히 추가/정리.  
**기대 효과:** 설치/운영 초기 실패 감소, 환경설정 오류로 인한 보안 사고 가능성 감소.
